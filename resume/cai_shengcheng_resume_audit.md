## 关键问题

### 1. 模糊量化，像「有数据但不敢写」

| 维度               | 内容                                                                                                                                                       |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **问题**     | 「推理性能达竞品**1.x 倍**」「开发效率提升**数倍**」反复出现，没有具体倍数、基准、测试条件                                                     |
| **影响**     | 资深面试官会怀疑：是 1.1 倍还是 1.9 倍？对标谁？什么 batch size / 序列长度？模糊表述会削弱`<1%` 精度误差的可信度                                         |
| **修改建议** | 能写就写死，例如「相对 FP16 baseline，Decode 吞吐提升 1.6x（batch=1, seq=4K，Ascend A5）」；不能公开就写占位符：`[吞吐提升待补：相对 FP16 约 1.5–1.8x]` |

## 修改策略

### 策略 A：摘要 — 从「组内介绍」改成「岗位卖点」

- **问题**：开头写「组内主要业务为…」，对面试官是噪音
- **扣分原因**：前 3 行没出现「推理性能 / 吞吐 / 延迟 / 引擎」等 JD 高频词
- **公式**：`[定位] + [年限] + [核心领域] + [最强成果]`

### 策略 B：量化 bullet — 把「1.x 倍」换成可验证指标

- **问题**：性能表述不可审计
- **扣分原因**：Inference 岗最看重 latency / throughput / memory footprint
- **原则**：每条量化 bullet 至少包含 **精度 + 性能 + 规模/周期** 三类中的两类；性能写不清就用占位符，不要写「1.x」

### 策略 C：对齐「大模型推理工程师」JD

- **问题**：简历 80% 是 PTQ/量化，20% 是推理引擎/调度
- **不改事实前提下的改法**：
  1. 技能区把「vllm等协同」展开为具体协作点，或降级为「了解 vLLM PagedAttention / Continuous Batching 原理」（**仅在你确实了解时写**）
  2. Wan2.2 改写成「**视频生成多模态大模型 W4A4 推理优化**」
  3. 面试准备话术与简历分工：简历写昇腾侧等价工作，面试讲方法论迁移

### 策略 D：删减低 ROI 内容

- **问题**：农机、过长调试工具描述占篇幅
- **改法**：一页版目标下，华为经历保留 **4–5 条** 高密度 bullet
