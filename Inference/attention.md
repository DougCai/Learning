# Prefill 与 Decode 阶段的 QKV 计算

以 prompt **「你是谁」** 为例，说明两阶段如何算 Q、K、V，以及 Decode 为何能复用 KV Cache。

---

## 基础：Q、K、V 是什么

每个字在某一层都有一条含义向量 $h_i$，通过线性变换拆成三份：

| 角色 | 含义 |
|------|------|
| **Q（Query）** | 这个字想找什么、关注什么 |
| **K（Key）** | 这个字能被怎么匹配 |
| **V（Value）** | 这个字携带的信息干货 |

公式：$Q_i = h_i W_Q$，$K_i = h_i W_K$，$V_i = h_i W_V$

注意力：用 $Q_i$ 对所有允许看的 $K_j$ 算匹配分数，再对 $V_j$ 加权求和。**因果规则**：位置 $i$ 只能看 $j \le i$，不能看后面。

---

## Prefill：整段 prompt 一起算

### 流程

1. **切 token**：「你」「是」「谁」→ 三个编号
2. **Embedding**：每个字变成含义向量 $h_1, h_2, h_3$
3. **每层算 QKV**：三个字 **一起** 算 $Q_1,K_1,V_1$、$Q_2,K_2,V_2$、$Q_3,K_3,V_3$
4. **注意力 + 写 Cache**：算完注意力后，把每个字的 **K、V 存入 KV Cache**
5. **预测首字**：站在「谁」后面，采样出第一个生成字（如「我」）

### 例子：「谁」怎么看整句

「谁」用 $Q_3$ 匹配 $K_1, K_2, K_3$，加权取 $V_1, V_2, V_3$。完整公式分三步：

**① 算匹配分数（点积 + 缩放）**

$$
s_{3,j} = \frac{Q_3 \cdot K_j}{\sqrt{d}}, \quad j \in \{1,2,3\}
$$

**② softmax 变成权重（和为 1）**

$$
\alpha_{3,j} = \frac{e^{s_{3,j}}}{e^{s_{3,1}} + e^{s_{3,2}} + e^{s_{3,3}}}
$$

**③ 对 V 加权求和，得到「谁」的输出**

$$
\text{out}_3 = \sum_{j=1}^{3} \alpha_{3,j} V_j = \alpha_{3,1} V_1 + \alpha_{3,2} V_2 + \alpha_{3,3} V_3
$$

写成一条就是 scaled dot-product attention（因果掩码下 $j$ 最大为 3）：

$$
\text{Attention}(Q_3, K, V) = \sum_{j=1}^{3} \text{softmax}\!\left(\frac{Q_3 K_j^\top}{\sqrt{d}}\right) V_j
$$

$\alpha_{3,j}$ 越大，说明「谁」越关注第 $j$ 个字的信息。综合后「谁」理解整句是在问 **你的身份**。

### Prefill 小结

| 项目 | 说明 |
|------|------|
| 输入 | 整段 prompt（3 个字） |
| 算 Q/K/V | **全部** prompt 位置都算 |
| KV Cache | **写入** 所有 prompt 的 K、V |

---

## Decode：每次只算 1 个新字

Prefill 后 Cache 里已有「你」「是」「谁」的 K、V。从生成「我」起进入 Decode。

### 流程（以生成「我」为例，序列变为「你是谁我」）

1. **只喂 1 个 token**：`「我」`
2. **只算 1 组 QKV**：$Q_4, K_4, V_4$（不重算前 3 个字）
3. **注意力**：用 $Q_4$ 对全部 $K_j$ 匹配、对 $V_j$ 加权（$j=1,2,3$ 从 Cache 读，$j=4$ 新算），见下式
4. **追加 Cache**：把 $K_4, V_4$ 写入 Cache 末尾
5. 下一步生成「叫」时，只算「叫」的 QKV，Cache 中「你～我」的 K、V 继续复用

### 例子：「我」怎么看前文

**① 匹配分数**（$K_1, K_2, K_3$ 从 Cache 读，$K_4$ 本步新算）

$$
s_{4,j} = \frac{Q_4 \cdot K_j}{\sqrt{d}}, \quad j \in \{1,2,3,4\}
$$

**② softmax 权重**

$$
\alpha_{4,j} = \frac{e^{s_{4,j}}}{\sum_{k=1}^{4} e^{s_{4,k}}}
$$

**③ 加权求和**（$V_1, V_2, V_3$ 从 Cache 读，$V_4$ 本步新算）

$$
\text{out}_4 = \sum_{j=1}^{4} \alpha_{4,j} V_j
$$

合并写法：

$$
\text{Attention}(Q_4, K_{\le 4}, V_{\le 4}) = \sum_{j=1}^{4} \text{softmax}\!\left(\frac{Q_4 K_j^\top}{\sqrt{d}}\right) V_j
$$

与 Prefill 公式形式相同，差别在于 **$j \le 3$ 的 $K_j, V_j$ 不再重算，直接从 KV Cache 读取**；只有 $K_4, V_4$ 是本步新算的。

### Decode 小结

| 项目 | 说明 |
|------|------|
| 输入 | **1 个**新 token |
| 算 Q/K/V | 只算 **当前字** |
| 历史 K、V | **从 Cache 读**，不重复算 |
| 新 K、V | 算完后 **追加** 进 Cache |

---

## 为什么 Decode 能复用 KV Cache

> **已经写进序列的字不会再变，它们对应的 K、V 也不会变。**

1. **历史 token 固定**：「你」「是」「谁」生成后不再修改，$K_i, V_i$ 只由当时输入决定，算一次结果永远相同。
2. **每步只有 1 个新字**：只需为新字算 $Q, K, V$；预测用的是 **当前字的 Q** 查 **全部历史的 K**。
3. **只 Cache K、V，不 Cache Q**：K、V 表示各位置的线索与内容，历史不变可复用；Q 表示「当前字想问什么」，每步不同，无需缓存。

**类比**：Prefill = 把问题整段读懂并归档；Decode = 每写一个新字只建自己的档案，回答时翻旧档案，不用把整句重新算一遍。

---

## 两阶段对比

| | Prefill（「你是谁」） | Decode（「我」「叫」…） |
|---|----------------------|-------------------------|
| 一次处理 | 整段 prompt | **1 个**新字 |
| 算 Q/K/V | 全部 prompt 位置 | 仅 **当前字** |
| KV Cache | **写入** | **读取**历史 + **追加**新 K/V |

---

## 一句话

- **Prefill**：整句一起算 QKV，因果注意力，K/V 写入 Cache，在「谁」后预测首 token。
- **Decode**：每步只算 1 个新字的 QKV，历史 K/V 从 Cache 读；能复用是因为过去的字及其 K/V 不再变化。
